6. Data Flow Details

Roster Update (Drag & Drop)
  1. @dnd-kit fires onDragEnd with handler ID + drop zone
  2. RosterPage determines new shift, calls executeShiftUpdate()
  3. Local state + localStorage updated
  4. syncData.updateRoster() emits via Socket.IO
  5. Server updates MongoDB: State.updateOne({key:'global'}, {$set:{roster}})
  6. Server broadcasts roster_updated to other clients
  7. Other clients update local state + localStorage
Name-Based Login
  1. User enters name + access key at Access Gate
  2. Socket connects; emits 'join' event
  3. Server validates access key (if configured)

JWT Account Login
  1. AuthContext.login() fetches CSRF token from GET /api/csrf-token
  2. POSTs to /api/login with username, password, CSRF header
  3. Server validates via bcrypt.compare(), generates JWT
  4. Sets httpOnly cookie 'token' (8h, sameSite varies by env)
  5. Returns user object { username, fullName, role }
  6. RoleContext calls /api/me for latest role
